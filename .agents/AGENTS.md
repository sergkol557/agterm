# Правила работы с проектом agterm

- **Документирование скриптов и Makefile-целей**: При добавлении любых новых вспомогательных скриптов в `scripts/` или новых целей в `Makefile` (например, для сборки, тестирования, деплоя или синхронизации), обязательно отражайте их описание в файле `CLAUDE.md` в секции `Build and test commands`.
- **Чистота рабочей директории при слиянии**: Скрипты синхронизации должны предварительно проверять статус репозитория на отсутствие незакоммиченных изменений, чтобы избежать случайной потери данных разработчика.

## Глобальные правила (macOS / Xcode)

- **Конфигурация архитектур сборки на macOS**: При создании или редактировании конфигурационных файлов сборки Xcode (таких как `project.yml` для `xcodegen` или настроек `.xcodeproj`), всегда используйте стандартные архитектуры `$(ARCHS_STANDARD)` вместо жестко прописанного значения `arm64` (если иное явно не указано пользователем). Это гарантирует совместимость с универсальными сборками macOS (Apple Silicon и Intel).
